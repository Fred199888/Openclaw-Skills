# narration-gen

## Instructions

1. 全程输出中文。
2. 接收参数：`article_text`（文章正文）、`output_path`（解说词文件路径，通常为项目的 `narration.txt`）。

### 解说词规范

**总体要求：**
- 口语化，像面对观众讲述
- 每段 40-90 字，适合单次 TTS 合成
- 用 `===` 作为段落分隔符
- 总段数 25-35 段（4-5 分钟视频）
- 目标时长：5 分钟以内

**叙事手法：场景代入 + 逐层递进**

核心原则：**用一个具体场景贯穿全片**，而非罗列技术点。

```
第 1 段：场景代入（"想象你正在用 XX，突然..."），让观众代入
第 2-3 段：点明问题和矛盾
第 4 段：揭示解决方案（一句话概括）
第 5-N 段：先讲传统做法的痛点（让观众理解「为什么需要」），再逐步展开新方案
倒数第 2 段：关键启示/总结
最后一段：结尾金句 + "谢谢观看"
```

**详细讲解的方式：**
- 核心概念用具体例子逐步走一遍（如一个请求经过系统的全过程）
- 每一步单独一段，不压缩到一句话里
- 对比讲解：先说传统做法（痛点），再说新做法（优势）
- 用「想象这个场景」引入关键决策时刻

**写作要诀：**
- 避免长句嵌套，每句 20 字以内
- 最后一段以总结金句结尾
- **杜绝废话和水词**：不要用"那这一切是怎么做到的呢？"、"接下来我们看看..."、"今天我们就来拆解..."这类纯过渡句，直接讲内容
- 过渡用一句话带过即可，不要单独占一段

**比喻和类比（提升记忆点的关键）：**
- 每个核心概念必须配一个生活化类比，帮助观众秒懂
- 好比喻的标准：听众不需要专业背景就能理解
- 类比模板：
  · 角色转变类："从XX变成了YY"（如"从泥瓦匠变成了系统设计师"）
  · 缺陷类比类："不是因为它笨，而是因为它瞎"
  · 场景类比类："就像在荒地上盖房子"（缺少基础设施的后果）
  · 对比反转类："传统做法=XX，新做法=YY"（如"代码=昂贵资产→代码=廉价产物"）
- 每 5-8 段至少一个精彩比喻
- 比喻要具体、画面感强，避免抽象空泛

**内容取舍（核心原则：重点详讲，已知略过）：**
- **详细讲解**：文章的核心创新点、关键架构设计、独特的解决思路 — 这些是观众看视频的理由。用多段展开，每段讲一个步骤或层级
- **简略带过**：业界常识、大家已知的背景知识 — 一句话交代即可
- **直接省略**：代码示例、冗长技术细节、引用来源、重复论述
- **不要**为了凑段数而注水，宁可短也不要啰嗦

**内容密度（每张幻灯片信息量要高）：**
- block 标题要言简意赅（2-6 字），描述补充具体内容
- 描述中可包含精辟引用（用引号标注，如"代码不再是资产，而是廉价产物"）
- 描述可包含对比（如"BEFORE: 手动编写 → AFTER: AI 生成+人工审核"）
- 结尾金句支持 `**关键词**` 标记（渲染时高亮），如"工程师从**泥瓦匠**变成了**系统设计师**"

### 幻灯片视觉分组（约 10 张视觉 × N 段字幕）

生成解说词的同时，按主题分组规划幻灯片视觉设计。

**核心原则：少跳转，多停留。**
- 全片约 **8-12 张**不同的视觉设计（不是每段音频换一张）
- 同一主题的多段音频共享同一张视觉，只有底部字幕随音频变化
- 每张视觉平均停留 20-40 秒，观感舒适不疲劳
- 最终 PNG 数量 = 音频段数（每个 PNG = 视觉 + 该段字幕）

在输出文件末尾添加注释（每行 = 一张视觉设计）：

```
# 幻灯片映射
# Slide 1 (封面): seg_01
# Slide 2 (问题背景): seg_02, seg_03
# Slide 3 (传统模式): seg_04, seg_05, seg_06, seg_07
# Slide 4 (混合模式): seg_08, seg_09, seg_10
# Slide 5 (决策瀑布): seg_11, seg_12, seg_13, seg_14, seg_15, seg_16, seg_17, seg_18
# ...
```

**注意：** 每个 Slide 行 = 一张视觉设计，seg 编号 = 共享该视觉的音频段。PPT 步骤会为每段音频生成独立 PNG（同一视觉 + 不同字幕），所以 PNG 总数 = 音频段数。

### 区块聚焦映射

在幻灯片映射之后，紧接着输出**区块聚焦映射**，用于 `slide_renderer.py` 的区块高亮渲染。

**格式：**

```
# 区块聚焦映射
# slide:1  | seg:01 | focus:none  | title:封面
# slide:2  | seg:02 | focus:block_1 | title:问题背景 | block_1:使用量远超预期 | block_2:核心矛盾
# slide:2  | seg:03 | focus:block_2 | title:问题背景 | block_1:使用量远超预期 | block_2:核心矛盾
# slide:3  | seg:04 | focus:all     | title:解决方案 | block_1:实时访问引擎 | block_2:无缝切换
# slide:10 | seg:31 | focus:none  | title:核心启示
# slide:10 | seg:32 | focus:none  | title:结尾
```

**格式规则：**
- `slide:N` = 视觉编号（同一 slide 共享相同布局，仅 focus 不同）
- `seg:MM` = 音频段编号（两位数，从 01 开始）
- `focus:block_K` = 高亮第 K 个区块（K 从 1 开始）
- `focus:none` = 无高亮（封面、结尾页）
- `focus:all` = 全部高亮
- `block_N:标题` = 区块简短标题（按编号顺序排列）
- `section:XX` = 英文分类标签，建议全大写（可选，如 `section:DIAGNOSIS & DEFINITION`）
- `icon_N:XX` = 第 N 个 block 的图标 emoji（可选，如 `icon_1:⚠️`）
- 同一 slide 的所有行必须共享相同的 title、blocks、section 和 icons，仅 focus 不同
- 每个 slide 最多 5 个 block
- 每段音频（seg）恰好对应一行映射
- 封面和结尾页无 block 定义，focus 为 none

**映射完整示例：**
```
# slide:2  | seg:02 | focus:block_1 | title:问题的提出 | section:DIAGNOSIS & DEFINITION | block_1:瓶颈不在模型 | icon_1:⚠️ | block_2:工程师角色转变 | icon_2:🔄
# slide:2  | seg:03 | focus:block_2 | title:问题的提出 | section:DIAGNOSIS & DEFINITION | block_1:瓶颈不在模型 | icon_1:⚠️ | block_2:工程师角色转变 | icon_2:🔄
```

**映射策略：**
- 同一 slide 的多段音频，依次聚焦不同区块（block_1 → block_2 → ...）
- 第一段可以 focus:all（整体概览），后续逐个聚焦
- 封面（slide:1）和结尾页始终 focus:none

### 输出

将解说词写入 `output_path`，格式：`段落\n===\n段落\n===\n...`

```
═══════════ narration-gen 完成 ═══════════
总段数：N
总字数：约 M 字
预估时长：约 X 分钟
文件：<output_path>
```
