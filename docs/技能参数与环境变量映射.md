# 技能参数与环境变量映射（可迁移版）

目标：把“机器相关参数”从脚本抽离，迁移到新电脑只改 env，不改代码。

## 一、变量分层（推荐）

### A. 全局环境变量（跨多个剧本共用）
建议文件：`~/.openclaw/env/global.env`
- XHS 鉴权：`XHS_COOKIE`, `XHS_X_S`, `XHS_X_S_COMMON`, `XHS_X_T`, `XHS_BRAND_USER_ID`, `XHS_TRACK_ID`
- 品牌通用：`XHS_COOPERATE_BRAND_ID`, `XHS_COOPERATE_BRAND_NAME`

### B. 剧本（skill）本地环境变量
建议文件：`.openclaw/skills/<skill>/.env`
- 表 ID / app token / 输出目录等“该 skill 特有参数”

## 二、已抽离的关键参数

### xhs-kol-crawl-all
- `XHS_KOL_APP_TOKEN`
- `XHS_KOL_TABLE_ID`
- `XHS_BACKUP_DIR`（默认相对路径 `./output/backups`）

### xhs-invite-batch
- `XHS_GLOBAL_ENV`（默认 `~/.openclaw/env/global.env`）
- `XHS_KOL_APP_TOKEN`
- `XHS_KOL_TABLE_ID`
- `XHS_INVITE_CONTENT`
- `XHS_COOPERATE_BRAND_ID`
- `XHS_COOPERATE_BRAND_NAME`

### kol-to-video-sync
- `XHS_KOL_APP_TOKEN`
- `XHS_KOL_TABLE_ID`
- `VIDEO_APP_TOKEN`
- `VIDEO_TABLE_ID`
- `KOL_VIDEO_BACKUP_DIR`（默认相对路径 `./output/backups`）

### kol-progress-to-video-sync
- `VIDEO_APP_TOKEN`
- `VIDEO_TABLE_ID`
- `SOURCE_TABLE_OPENCLAW_BD`
- `SOURCE_TABLE_INFO_BD`
- `KOL_VIDEO_BACKUP_DIR`

## 三、路径策略（已改）
- 备份输出从“绝对路径”改成“相对路径 + 环境变量可覆盖”。
- 这样在别人电脑上只需改 `.env`，无需改脚本。

## 四、迁移建议
1. 复制 `.openclaw/skills/*`
2. 复制 `.openclaw/env/global.env.example` 并改成 `global.env`
3. 按每个 skill 的 `.env.example` 填值
4. 如有 cron，检查任务 message 中是否有旧机器绝对路径，改成目标机路径
